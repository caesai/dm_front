## CertificatesSelector
Компонент селектора, который позволяет пользователям выбирать между онлайн и оффлайн сертификатами.

Компонент предназначен для использования на странице бронирований(BookingPage, BookingFreeEventPage, BookingRestaurantPage), который предоставляет необходимые пропсы (setCertificateId, isOpened, selectedCertificateId).

### API (Props)
Компонент принимает следующие свойства:

| Название         | Тип                          | Описание       | Обязательный |
|------------------|------------------------------|----------------|-------------|
| setCertificateId | (id: string \| null) => void | Функция обратного вызова для обновления выбранного ID сертификата в родительском компоненте. | Да |
| isOpened         | Boolean                      | Флаг, указывающий, отображается ли в данный момент селектор (используется для синхронизации состояния при открытии UI).| Да          |
| selectedCertificateId         | string \| null                      | Текущий ID сертификата, переданный из родительского состояния.          | Да          |

### Логика и Поведение

1. **Загрузка данных:**  Компонент читает список доступных сертификатов из Jotai-атома certificatesListAtom. Если список пуст, компонент не рендерится (return null).
2. **Синхронизация при открытии:** При изменении isOpened на true и наличии selectedCertificateId, компонент автоматически синхронизирует свое внутреннее состояние, выбирая соответствующий сертификат и его тип.
3. **Выбор типа:** Пользователь может переключаться между типами сертификатов (онлайн/оффлайн), что сбрасывает предыдущие выборы при необходимости.
4. **Синхронизация выбора:** При выборе конкретного онлайн-сертификата внутренний useEffect немедленно вызывает setCertificateId(id), обновляя родительское состояние.
